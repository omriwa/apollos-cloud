Microservices Architecture & AWS Backend for Vivy HealthTech

Vivy, a Berlin-based digital health startup backed by Allianz, required rapid development of their healthcare platform MVP. The challenge was to build a modern, modular frontend and backend architecture that could seamlessly integrate with their existing healthcare ecosystem while meeting strict German healthcare data regulations.

Context

The Vivy health app aimed to give users control over their medical records through a secure digital platform. The project required building a micro-frontend architecture alongside a robust AWS backend that could serve as both a development mockup server and a production-ready API layer.

I contributed to the core architecture and development of the platform, focusing on the Node.js backend, GraphQL API design, and micro-frontend integration patterns.

Engineering Challenges & Solutions

Micro-Frontend Architecture

The platform required independently deployable frontend modules that could be developed by separate teams. We implemented a micro-frontend pattern enabling modular UI composition while maintaining a cohesive user experience.

GraphQL API Design

Healthcare data access patterns are complex and varied. We designed a GraphQL API layer that provided flexible data querying for the frontend while abstracting the complexity of multiple backend services and data sources.

Backend Mockup Server

To enable parallel frontend and backend development, I built an AWS-based mockup server that accurately simulated the production API behavior, allowing frontend teams to develop and test without backend dependencies.

The Architectural Solutions

Node.js Backend Platform

Built a Node.js backend with clean service architecture, providing RESTful and GraphQL endpoints for the healthcare data platform.

GraphQL Data Layer

Designed and implemented a GraphQL API that unified access to medical records, user profiles, and healthcare provider data.

AWS Infrastructure

Deployed on AWS with proper security configurations for healthcare data, including encryption at rest and in transit.

Micro-Frontend Integration

Implemented micro-frontend composition patterns allowing independent team development with shared design system.

Technical Impact

Rapid MVP Delivery

Delivered a working MVP with full frontend-backend integration within tight startup timelines.

Parallel Development

The mockup server enabled frontend and backend teams to work independently, doubling effective development velocity.

Modular Architecture

Micro-frontend approach allowed the platform to scale its development team without architectural bottlenecks.
