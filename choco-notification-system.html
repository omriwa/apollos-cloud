Scalable Client Notification Infrastructure: AWS CDK & Step Functions for Choco

Choco (PaaS, Germany) required a highly reliable and scalable client notification system to handle mission-critical communications. The objective was to build an internal monitoring and alerting system that ensured 100% deliverability and low latency while integrating with third-party messaging providers like Twilio and WhatsApp.

Scalable Client Notification Infrastructure: AWS CDK & Step Functions for Choco

Context

The Choco Notification Engine is a high-throughput backend service designed to orchestrate complex communication workflows across multiple messaging channels.

Working within the backend team and collaborating closely with DevOps and the Lead Architect, I led end-to-end projects, managing both business and technical aspects. The architecture utilized serverless AWS components and a state-machine-driven logic based on automata to ensure mission-critical reliability.

Deep Engineering Challenges

Complex Workflow Management

Implementing reliable notification flows across different providers required a robust state-management solution. By utilizing AWS Step Functions, we replaced manual processes with a maintainable state machine that handled complex business logic and transmission scenarios.

Latency Optimization

To meet strict service requirements, the system needed to process messages with minimal delay. We optimized the serverless components, resulting in a 50% reduction in notification latency.

Advanced IaC Scaling

Using AWS CDK, I designed the infrastructure to be modular and environment-aware, enabling the team to manage identical stacks for multiple environments (development, staging, live) seamlessly.

SLA Compliance & Monitoring

Ensuring high deliverability required a detailed monitoring layer. I implemented custom CloudWatch Alarms and metrics to monitor system performance and ensure SLA compliance.

The Architectural Solutions

Under the leadership of Omri Wallach, Apollo's Cloud engineered a robust notification backbone using Infrastructure as Code (IaC) and modern asynchronous patterns.

State Machine Orchestration

Developed business logic using AWS Step Functions to manage complex, multi-step notification workflows and automata-based logic.

Infrastructure as Code (IaC)

Fully automated the provisioning of AWS resources using AWS CDK, ensuring repeatable and version-controlled environments.

Multi-Channel Integration

Integrated API Gateway and Lambda with Twilio and WhatsApp messaging providers for global reach.

Internal Monitoring & Alerting

Built a comprehensive monitoring system using CloudWatch metrics and alarms to track system health in real-time.

Technical Impact & Conclusions

The transition to a CDK-managed, state-machine-driven architecture provided Choco with a professional-grade communication layer that directly improved user experience and operational transparency.

50% Latency Reduction

Performance tuning and architectural refactoring boosted SLA compliance and message speed.

Enhanced Reliability

The state machine approach ensured robust transmission through internal microservices and third-party APIs.

Increased Codebase Quality

Implemented linters, compilers, and pre-commit hooks (Husky) to maintain high development standards.

Accelerated Deployment

Standardized CI/CD with GitHub Actions for rapid, reliable releases.

Technical Stack

AWS CDK AWS Step Functions AWS Lambda API Gateway CloudWatch Twilio WhatsApp API Node.js GitHub Actions Serverless Architecture State Machines
