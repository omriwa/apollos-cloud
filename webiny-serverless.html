Serverless Infrastructure & Pulumi IaC for Webiny CMS

Serverless infrastructure with Pulumi IaC and GraphQL API for Webiny open-source headless CMS.

Context

Webiny is a headless CMS on serverless AWS. I contributed to Pulumi infrastructure, GraphQL resolvers, and serverless optimization.

Engineering Challenges

IaC at Scale

Pulumi (TypeScript) modular enough for user customization across hundreds of AWS accounts.

Serverless Optimization

Lambda, DynamoDB, S3 cold start optimization and connection pooling.

GraphQL API

Complex GraphQL resolvers with content modeling and multi-tenancy.

Architectural Solutions

Pulumi Infrastructure

TypeScript modules for Lambda, API Gateway, CloudFront, DynamoDB.

GraphQL API Layer

Resolvers across DynamoDB and Elasticsearch.

Serverless Optimization

Lambda cold start and DynamoDB access pattern optimization.

Open Source

Code reviews, docs, features following community standards.

Measurable Impact

Reliability

More reliable deployments across diverse AWS environments.

Performance

Reduced API response times for content delivery.

Developer Experience

Easier deployment and customization for community.
